# Angular + Stencil

Angular Wrappers: Similarly to Vue and React, an Angular wrapper provides an Angular interface to a Stencil web component, making the web component feel more like a typical Angular component. This includes proper handling of props, state, and events within the context of an Angular application.

## Installation

**1.**

**- with NPM**

`npm install @infineon/infineon-design-system-angular`

**- with Yarn**

`yarn add @infineon/infineon-design-system-angular`

**2. Installation of SASS**

```bash
npm install sass
```

**3. Import the module inside your entry point file**

```js
import { defineCustomElements } from "@infineon/infineon-design-system-stencil/loader";

defineCustomElements(window);
```

**4. Additional steps**

**Create app.modules.ts file inside your app folder, and copy inside the following code**:

```js
import { CUSTOM_ELEMENTS_SCHEMA } from '@angular/core';

@NgModule({
 ...
  schemas: [
    CUSTOM_ELEMENTS_SCHEMA
  ],
 ...
})
```

**Update app.ts file:**:

```js
import { Component, CUSTOM_ELEMENTS_SCHEMA } from '@angular/core';

@Component({
  ...
  schemas: [CUSTOM_ELEMENTS_SCHEMA]
})
```

## Usage

```js

<ifx-button disabled="false" variant="primary" size="m" target="_blank" theme="default" >
  Button 
</ifx-button>
```

**Note**: In Angular, you do not have to import each component before using them.

### Events

#### Two way data binding

Two-way binding means that any data-related changes affecting the model are immediately propagated to the matching view(s).
And any changes made in the view(s) are immediately reflected in the underlying model.
When app data changes, so does the UI, and conversely.

```js
//app.ts
switchChecked = false;
switchDisabled = false;
...
toggleSwitchValue(event: any) {
  console.log("switch value change")
  this.switchChecked = event.detail;
}
//app.html
<ifx-<ifx-switch [disabled]="switchDisabled" [value]="switchChecked" (ifxChange)="toggleSwitchValue($event)"></ifx-switch>
```
