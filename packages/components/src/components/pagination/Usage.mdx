# Pagination
The pagination provides a user-friendly way to navigate through a set of data or content that is divided into multiple pages. It allows users to easily move between pages and access the desired information.

## Usage

### When to use
- When presenting a large dataset or content that needs to be divided into multiple pages like tables or product lists.

### When not to use
- When the data or content is small enough to fit comfortably on a single page without causing visual clutter.
- If the content is best consumed in a continuous or uninterrupted flow, such as articles or long-form text.

## Behaviour

### Desktop
<img src={PaginationDesktop} alt="Pagination Desktop" />

- The maximum number of visible page elements at any given time is limited to 7. If more pages need to be displayed, all additional elements are truncated. 
- The last number of pages is always displayed at the last position of the pagination to ensure that users are aware of all the content available.

### Mobile
<img src={PaginationMobile} alt="Pagination Mobile" />
- Similar to the desktop version, the mobile pagination will display a maximum of 5 page items at a time.

## Anatomy
<img src={PaginationMoreAnatomy} alt="Pagination Anatomy" />

**Results per page (1)**\
Utilize the select component to enable users to specify the number of results displayed per page.


**Navigational Controls (2)**\
Users are enabled to navigate through the available pages using controls that inherit styling and interactions from our Secondary Icon Button.


**Current page (3)**\
Shows the current active page.


**Truncation (4)**\
The hidden or less relevant pages are displayed with an ellipsis (...)


**Maximum Page (5)**\
Provides a clear endpoint, allowing users to understand the full scope of available content.

## UI Examples

### Desktop
<img src={PaginationPositioningDesktop} alt="Pagination Positioning Desktop" />
- The pagination is aligned at the bottom right of the parent container. 
- Users have the option to select the number of results they want to display per page via a dropdown menu.

### Mobile
<img src={PaginationPositioningMobile} alt="Pagination Positioning Mobile" />
- In the mobile version the select option for choosing the number of results per page is disabled.
- The mobile pagination will be center-aligned within its parent container, and it dynamically takes up 100% of the grid container's width.
