# Chip
The chip component presents compact information in an easily consumable format. Depending on the context it is used in, a chip can act as a purely static display element or as an interactive control. This flexibility — combined with different variants, styles, and sizes — allows the chip to adapt to a wide range of use cases across the interface.

## Usage

### When to use
- If you want to display metadata, tags, or labels in a compact, readable format.
- If you want to filter and categorize content.
- If you want to allow users to add or remove options from a predefined set.
- If you need a flexible element that can serve as either a static indicator or an interactive control depending on context.

### When not to use
- If the content within the chip is dynamically changing or frequently updated.
- If you need to display a large amount of complex data or detailed information. Use other components like tables or lists.
- If you need to communicate a status or state such as "Active", "In Review", or "Pending". Use the Status component instead.
- If the action behind the chip is a primary call-to-action. Use a Button component instead.

## Static vs. Interactive

The Default chip can be used in two fundamentally different ways depending on the context it is placed in.

**Static chips** serve as non-interactive display elements. They communicate information — such as a status, tag, or category — without expecting any user input. Static chips do not respond to hover or focus interactions and cannot be clicked.

<img src={ChipStaticExample} alt="Example of static chip usage" />

**Interactive chips** respond to user input. They can be selected, toggled, or dismissed, and trigger actions or filter changes within the interface. Interactive chips expose hover, focus, pressed, and selected states.

The Option chip is always interactive by definition, as it always opens a select list on click.

## Anatomy

<img src={ChipAnatomy} alt="Chip Anatomy" />

**Chip Container (1)**\
The entire chip, including its label, icons, and any additional styling, is contained within a visually distinct container.

**Dropdown Icon (2)**\
Indicates that there are additional options related to the chip. Clicking or tapping on the dropdown icon reveals a select list with different options to choose from. Only present on Option chips.

**Label Text (3)**\
This is the primary text that describes the chip or the category being filtered. We recommend keeping labels short — ideally 2 to 3 words. Labels are truncated after 25 characters.

**Option Text (4)**\
In the Option chip variant, the currently selected option is shown inline next to the label.

**Number Indicator (5)**\
Provides information about how many options are selected inside the chip. It appears when more than one option has been selected in a multi-select context.

**Dismiss Icon (6)**\
The dismiss icon allows users to reset their selection with one click and return the chip to its default state. Only present on removable Default chips and active Option multi chips.

## Variants

### Default Chip
The Default chip is a versatile element that can represent tags, labels, filter tokens, or status indicators. It can be configured as either a static or interactive element depending on the context. An optional left-side icon and a dismiss icon for removal are available.

<img src={ChipDefault} alt="Chip Default" />

#### States
The states below apply when the Default chip is used as an **interactive** element. Static chips do not exhibit hover, focus, or pressed states.

#### Icon configurations

<img src={ChipDefaultIconVariants} alt="Chip Icon Variants" />

The following icon configurations are available:
- **Label only** — plain text chip with no icon
- **Left icon** — icon placed before the label text
- **Right icon** — icon placed after the label text
- **Left icon + Right icon** — both icons are present simultaneously

#### Filled light
The filled light style uses a light-colored fill as its default appearance.

### Option Chip
By clicking on the Option chip, a select list is displayed. This gives users the possibility to filter content in more detail and reach desired results more quickly. The Option chip is **always interactive** and is available in two types: single select and multi select.

<img src={ChipOptionDefault} alt="Chip Option Default" />

#### States
<img src={ChipOptionStates} alt="Option Chip States" />

#### Single
This type uses a single select list. The selected option is shown inline within the chip, next to the label text.

<img src={ChipOptionSingleDropdown} alt="Option Chip Single Dropdown" />

#### Multi
This type uses a multi select list. Multiple options can be selected simultaneously. Once more than one option is selected, a number indicator shows the count of additional selections. A dismiss icon is displayed to allow resetting all selections at once.

## Chip Group
In most cases, the chip component is used as a chip group, which also exists as its own component. Chips within a group wrap automatically onto new rows when the container width is exceeded. A chip group can contain static chips, interactive chips, or a combination of both — as long as the intended behavior is communicated consistently through visual styling.

<img src={ChipGroup} alt="Chip Group" />

## Size
There are three available sizes: small, medium (default), and large. The appropriate size should be chosen based on the surrounding layout and context — for example, smaller chips work well in dense data views, while larger chips are better suited for touch interfaces or more prominent filter areas.

<img src={ChipSizes} alt="Chip Sizes" />

## Mouse interaction
Depending on which chip variant and context is used, there are different clickable areas. Static chips have no clickable area. Interactive Default chips in their default state are fully clickable as a whole. Chips with a dismiss icon consist of two distinct clickable areas: clicking the dismiss icon resets the chip to its default state, while clicking the remaining chip area triggers its primary interaction — such as toggling a selection or opening a select list.
