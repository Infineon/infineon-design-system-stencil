# Status
The status component serves as a visual indicator to convey additional information about an element on the interface. It is often used to highlight the status of processes to enhance the user's understanding of the content.

## Usage

### When to use
- If you want to visually highlight past, current or upcoming events.
- If you want to provide supplementary information to a product inside a table.

### When not to use
- Avoid using status components for elements that do not carry significant importance or meaning.
- Do not use status components solely for decorative purposes, especially if they do not serve a functional or informative role.
- Avoid using status components to display redundant or duplicative information.
- Avoid to use long label names. Be precise and concise.

In case you need more status colors, please refer to our [Color Foundations](https://infineon.github.io/infineon-design-system-stencil/?path=/docs/foundations-colors--development).

### Project status
We recommend the following color coding for the most common status conditions:

Error = `$ifxColorRed500`

In Progress = `$ifxColorOcean500`

Completed = `$ifxColorLawn500`

Blocked = `$ifxColorSun500`

### Semantic coloring

To reinforce semantic relationships, use similar colors for related items. For example apply `$ifxColorRed500` , `$ifxColorRed600` and `$ifxColorRed700` for different error states to maintain hue consistency.

### User activity
We recommend the following color coding for the most common user activities:

Not available = `$ifxColorEngineering200`

Active = `$ifxColorOcean400`

Blocked = `$ifxColorRed500`
