# Status
The status component serves as a visual indicator to convey additional information about an element on the interface. It is often used to highlight the status of processes to enhance the user's understanding of the content.

## Usage

### When to use
- If you want to visually highlight past, current or upcoming events.
- If you want to provide supplementary information to a product inside a table.

### When not to use
- Avoid using status components for elements that do not carry significant importance or meaning.
- Do not use status components solely for decorative purposes, especially if they do not serve a functional or informative role.
- Avoid using status components to display redundant or duplicative information.
- Avoid to use long label names. Be precise and concise.

## Use case examples

> In general, we recommend the use of clearly different-looking / high-contrast colours, e.g. to differentiate the priority of topics.

### Project status
Colors can indicate the status of tasks. For example you can use:
`red-500` for Blocked, `orange-500` for In Progress, `engineering-300` for Not Started and `green-500` for Done.

> We recommend the use of colours and their associated nuances, e.g. for neutral and restrained contexts such as marking the activity of a user.

### User activity
Colors can indicate the activity of users. For example you can use:
`engineering-200` for Not available, `engineering-400` for Active but busy, `engineering-600` for Blocked.

Also check out the infineon [Colors Foundation →](https://infineon.github.io/infineon-design-system-stencil/?path=/docs/foundations-colors--development)
